# Testing Types

## 1. Regression Testing

Regression testing is the process of testing a software application to ensure that new changes  
(like updates, bug fixes, or new features) do not negatively impact the existing functionality of  
the software.

### Types of Regression Testing

- **Unit Regression Testing**  
  We test only the specific changes made by the developer.

- **Regional Regression Testing**  
  We test the part that was changed along with the connected parts.

- **Full Regression Testing**  
  We test the main part that was changed and also check the rest of the software to be thorough.

---

## 2. Re-Testing

Whenever the developer fixes a bug, the tester tests whether the bug fix is working correctly.  
This is called **Re-testing**.

---

## 3. Exploratory Testing

- We have to explore the application, understand it completely, and test it.
- Understand the application, identify all possible scenarios, document them, and then use them for testing.
- Exploratory testing is done when the application is ready but there is no requirement document.

---

## 4. Positive and Negative Testing

- **Positive Testing**:  
  Testing the application with valid inputs to verify expected behavior.

- **Negative Testing**:  
  Testing the application with invalid or unexpected inputs to check error handling.

---

## 5. End-to-End Testing

End-to-end testing verifies the complete flow of the application from start to finish,  
ensuring all integrated components work together as expected.

---

## 6. Globalization & Localization Testing

- **Globalization Testing**  
  Ensures the application works across different regions, languages, and cultures without code changes.

- **Localization Testing**  
  Ensures the application is adapted correctly for a specific language or region (date, time, currency, language, etc.).
