## 1. Software Testing

**Software Testing** is the process of checking an application to ensure it works **as expected** and is **free from defects (bugs)**.

In testing, we mainly focus on:

*   **What to test** in an application
*   **How to test** it
*   **Whether it meets user and business requirements**

The goal is to **identify defects early** and deliver a **quality product**.

***

## 2. Types of Software

Software is broadly divided into **three categories**:

***

### 1) System Software

System software manages and controls the computer hardware and provides a platform for other software.

**Examples:**

*   Operating Systems (Windows, Linux, macOS)
*   Device Drivers (printer driver, keyboard driver)
*   Servers
*   Utilities (antivirus, disk cleanup tools)

✅ **Testing focus**:

*   Performance
*   Security
*   Compatibility with hardware
*   Stability under heavy load

***

### 2) Programming Software

Programming software helps developers **create, test, and maintain programs**.

**Examples:**

*   Compilers (C, Java compiler)
*   Debuggers
*   Interpreters (Python interpreter)
*   IDEs (Eclipse, VS Code)

✅ **Testing focus**:

*   Correct error detection
*   Accurate code translation
*   Performance and reliability

***

### 3) Application Software

Application software is designed for **end users** to perform specific tasks.

**Examples:**

*   Web Applications (Amazon, Flipkart)
*   Mobile Apps (WhatsApp, Instagram)
*   Desktop Applications (MS Word, Excel)

✅ **Testing focus** (very important for testers):

*   Functionality
*   UI/UX
*   Usability
*   Security
*   Compatibility

***

## 3. Software Quality

**Software Quality** means how well the software satisfies **user needs and expectations**.

A **high‑quality software** has the following characteristics:

***

### 1) Bug‑free

*   The application works **without errors or crashes**
*   All features behave as expected

✅ Example:  
Login works with valid credentials and shows error for invalid ones.

***

### 2) Delivered on Time

*   The software is released **within the planned schedule**
*   Delays increase cost and customer dissatisfaction

✅ Example:  
A banking app released before the financial year ends.

***

### 3) Within Budget

*   Development and testing costs are **controlled**
*   No unnecessary rework due to late bug discovery

✅ Example:  
Finding bugs early in testing saves money.

***

### 4) Meets Requirements and/or Expectations

*   Software behaves exactly as mentioned in the **requirement documents**
*   Satisfies both **business needs** and **user expectations**

✅ Example:  
If requirement says “OTP should expire in 2 minutes”, it must do so.

***

### 5) Maintainable

*   Easy to **modify, fix, and enhance** in the future
*   Well‑structured code and documentation

✅ Example:  
Adding a new feature without breaking existing functionality.

***

## 4. Role of Testing in Software Quality

Testing ensures:

*   Bugs are detected early
*   Requirements are met
*   User gets a reliable product
*   Overall software quality is maintained
