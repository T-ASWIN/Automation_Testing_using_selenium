# Non-Functional Testing

## Types of Non-Functional Testing

### 1. Performance Testing
Performance testing checks the speed, stability, and scalability of the application.

- **Load Testing**  
  Checks application behavior under expected user load.

- **Stress Testing**  
  Checks application behavior under extreme load conditions.

- **Endurance Testing**  
  Verifies system performance over a long period of time.

- **Spike Testing**  
  Checks application response to sudden increase or decrease in load.

- **Volume Testing**  
  Verifies system behavior with large volumes of data.

---

### 2. Security Testing

Ensures the application is protected against unauthorized access.

- **Authentication**  
  Verifies user identity (login validation).

- **Authorization**  
  Verifies user permissions and access rights.

---

### 3. Recovery Testing

Recovery Testing assesses how well a system can recover after a failure or crash.  
It checks whether the software can resume normal operations without data loss.

---

### 4. Compatibility Testing

Ensures application works properly in different environments.

- Operating System compatibility  
- Browser compatibility (Cross-browser testing)  
- Hardware compatibility (Configuration testing)

---

### 8. Configuration Testing

Validates application behavior across different system configurations such as:
- Hardware
- Software
- Network settings

---

### 9. Installation Testing

Verifies whether the application is installed, upgraded, or uninstalled successfully on different systems.

---

### 10. Sanitation / Garbage Testing

Checks application behavior with invalid, random, or unexpected inputs to ensure stability.
